## Overview

An analytical machine learning project focused on comparing multiple classical algorithms to identify the optimal approach for predicting heart disease based on patient data.

## Problem

In healthcare analytics, selecting the right machine learning model is as critical as the prediction itself. Different algorithms have varying strengths regarding interpretability, computational efficiency, and accuracy, making model selection a non-trivial challenge.

## Why This Approach ⭐

Rather than blindly deploying the most complex model, this project prioritized a **Comparative Evaluation Framework**. By testing Logistic Regression, K-Nearest Neighbors, Decision Trees, and Random Forests side-by-side, the project provided a clear view of model performance trade-offs.

## Solution + Architecture

- **Data Processing**: Python and Pandas for cleaning and feature engineering patient health data.
- **Algorithms Evaluated**: LR, KNN, Decision Tree, Random Forest using Scikit-learn.
- **Evaluation Strategy**: Focused on accuracy, precision, recall, and computational efficiency to find the best real-world fit.

## Results / Impact

- Achieved **~86% overall accuracy** across the optimal models.
- Established a robust methodology for classical machine learning model evaluation and comparison.

## Key Insights ⭐

- **Ensemble Methods Win:** Random Forest generally provided the highest accuracy by overcoming the overfitting tendencies of standard Decision Trees.
- **Interpretability vs. Accuracy:** In medical contexts, simpler models like Logistic Regression sometimes offer better interpretability, which is highly valued by clinical end-users, even if accuracy is marginally lower.
