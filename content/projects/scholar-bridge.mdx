## Overview

Scholar Bridge is a centralized, full-stack web application developed to improve communication and resource sharing across different stakeholders within an educational institution.

## Problem

Academic institutions often suffer from fragmented communication channels. Important notices, resources, and interactions between students, Teachers, and Heads of Departments (HODs) are typically scattered across emails, physical notice boards, and unstructured messaging apps.

## Why This Approach ⭐

A monolithic, role-based web architecture was chosen to ensure a single source of truth. By building a custom Django backend, the system could enforce strict role-based access control (RBAC), ensuring that students, teachers, and administrators had appropriate permissions and tailored dashboards.

## Solution + Architecture

- **Backend**: Python and Django for robust server-side logic and ORM.
- **Database**: MySQL for structured, relational data storage.
- **Frontend**: HTML, CSS, and vanilla JavaScript.
- **Features**: Multi-role authentication, notice broadcasting, and centralized resource sharing.

## Results / Impact

- Successfully enabled seamless multi-role interaction (Students, HODs, Teachers) within a single unified platform.
- Demonstrated early proficiency in full-stack system design, database modeling, and secure web application development.
